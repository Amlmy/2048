## 2048 小游戏

### 1\. 项目目标

使用 Python 编程语言，结合图形用户界面（GUI）库，实现一个功能完整、界面友好、可交互的经典 2048 小游戏。

### 2\. 用户群体

  * 喜爱益智游戏的休闲玩家。
  * Python 学习者和开发者。

### 3\. 功能需求 (Functional Requirements)

#### 3.1 游戏核心逻辑 (Game Core Logic)

| 编号 | 需求名称 | 详细描述 |
| :--- | :--- | :--- |
| FR1.1 | **初始化棋盘** | 游戏开始时，生成一个 $4 \times 4$ 的网格作为棋盘，并在随机两个空位上生成数字 '2' 或 '4'（通常是 '2' 的概率更高）。 |
| FR1.2 | **数字生成** | 每执行一次有效的移动操作后，在棋盘上的一个随机空位上生成一个新的数字 '2' 或 '4'。 |
| FR1.3 | **移动操作** | 玩家可以通过键盘输入（上下左右）来移动棋盘上的所有数字。**移动规则：** 数字向指定方向移动到最远处的空位。 |
| FR1.4 | **合并操作** | 在移动过程中，如果两个相邻且相等的数字相撞，它们将合并成一个新的数字（原数字的两倍），并在一次移动中只进行一次合并。 |
| FR1.5 | **得分计算** | 每次发生合并时，将合并后的新数字的值加到总分中。 |
| FR1.6 | **胜利判定** | 当棋盘上出现数字 '2048' 时，游戏胜利。游戏可以继续进行。 |
| FR1.7 | **失败判定** | 当棋盘上所有位置都被数字填满，并且任何方向（上、下、左、右）都无法进行有效的移动和合并操作时，游戏失败（Game Over）。 |

#### 3.2 图形用户界面 (GUI)

| 编号 | 需求名称 | 详细描述 |
| :--- | :--- | :--- |
| FR2.1 | **主界面显示** | 界面应清晰显示 $4 \times 4$ 的游戏棋盘。 |
| FR2.2 | **方块绘制** | 棋盘上的每个数字方块应以不同的背景颜色或样式区分，以便玩家快速识别其值。 |
| FR2.3 | **分数显示** | 实时显示当前游戏得分（Current Score）。 |
| FR2.4 | **最高分显示** | 显示历史最高得分（Best Score），并具备保存功能（可选：通过文件存储）。 |
| FR2.5 | **游戏状态提示** | 胜利或失败时，应以显著方式（如弹窗或覆盖层）给出提示信息。 |
| FR2.6 | **重开按钮** | 提供一个按钮或快捷键，允许玩家随时重新开始一局新游戏。 |
| FR2.7 | **撤回按钮** | 提供一个撤回按钮，允许玩家随时撤回到上一個步骤。 |

#### 3.3 用户交互 (User Interaction)

| 编号 | 需求名称 | 详细描述 |
| :--- | :--- | :--- |
| FR3.1 | **键盘控制** | 游戏应能响应键盘的 '上', '下', '左', '右' 键（或 'W', 'S', 'A', 'D' 键）进行移动操作。 |
| FR3.2 | **界面更新** | 每次有效移动后，棋盘和分数应立即在 GUI 上更新。 |
